# Typed API client (OpenAPI → TypeScript)

The Go server's Huma operations generate an OpenAPI 3.1 spec. We commit that
spec **and** the typed TS client derived from it so PRs surface API-surface
changes loudly.

## Regenerating

```bash
make gen-api-client          # regenerate web/shared/api.openapi.json + types
./bin/heya openapi-spec      # dump just the spec (use -o file to write)
```

The lefthook `openapi-drift` hook regenerates the client whenever a
`*_huma.go` file changes and blocks the commit if the regenerated artifacts
differ from what's staged — equivalent to the sqlc drift check.

## Files

| File / alias                          | Role                                                                          |
| ------------------------------------- | ----------------------------------------------------------------------------- |
| `web/shared/api.openapi.json`         | Full spec, regenerated by the Make target. Consumed by `nuxt-open-fetch` at Nuxt build time; no separate TS-codegen step. |
| `#open-fetch-schemas/heya`            | Auto-imported alias exposing `paths`, `components`, `operations` derived from the spec. Use `components['schemas']['Foo']` for type aliases outside endpoint calls (example: `web/app/pages/settings/server.vue`). |
| `web/app/plugins/heyaApi.client.ts`   | Wires the bearer token + 401-logout via `nuxt-open-fetch` hooks. No call-site boilerplate needed. |

## Calling the API

The Nuxt module `nuxt-open-fetch` generates two typed clients off the spec:

- **`useHeya('/api/path', opts)`** — reactive `useFetch` wrapper. Returns
  `{ data, pending, error, refresh }`. SSR-capable. Use in `<script setup>`
  for page-level loads.
- **`$heya('/api/path', opts)`** — imperative `$fetch` wrapper. Use in event
  handlers, mutations, composable functions. Access via
  `const { $heya } = useNuxtApp()`.

Both infer path params, query, body, and response from the spec. Path params
go in `path: { ... }`, query in `query: { ... }`. Drop `Content-Type` headers
and `JSON.stringify` — handled.

## When to keep raw `$fetch`

Skip these endpoints (use raw `$fetch`):

- Binary streams: `/api/media/{id}/image/*`, `/api/stream/*` segments,
  `/api/tracks/{id}/stream`, `/api/person/{id}/image`
- Multipart uploads: `/api/media/{id}/assets/upload`
- WebSocket and SSE

## Reference call sites

- `web/app/pages/settings/server.vue` — reactive + types
- `web/app/components/library/LibrarySidebar.vue` — imperative
- `web/app/composables/usePlaylists.ts` — composable pattern

## Gotchas

- **Spec-vs-app type mismatches**: cast through `unknown` (`as unknown as T`).
- **`/api/media/{id}` accepts both slug and numeric IDs** but the spec types
  it as `string`, so wrap numeric IDs with `String(...)`.
- **`useNuxtApp()` placement**: see the
  [UI conventions](./ui.md#never-call-usenuxtapp-inside-computed-or-async-bodies)
  — hoist `const { $heya } = useNuxtApp()` to script-setup top level, never
  inside `computed` or async bodies.
